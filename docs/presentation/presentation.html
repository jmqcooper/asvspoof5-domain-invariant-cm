Research Questions & Hypotheses

Main RQ: Do speech deepfake detectors use codec information as a shortcut, and if so, can DANN mitigate shortcut learning?

Sub-Questions

  • RQ1: Does DANN improve generalization under codec shift vs ERM?
  • RQ2: What is the trade-off between detection and domain invariance?
  • RQ3: Where is codec information encoded, and how does DANN change it?
  • RQ4: Can activation patching reduce domain leakage without retraining?

Hypotheses

  • H1: DANN reduces domain leakage and improves out-of-domain EER
  • H2: Domain information concentrates in specific layers; DANN suppresses it
  • H3: Effects are consistent across WavLM and Wav2Vec2, but layer-wise localization differs

The Problem: Domain Shift

Detectors Fail Under Real Conditions

  • Models trained on clean data degrade when audio is compressed, transmitted, or re-recorded
  • Detectors learn shortcut cues tied to codec artifacts
  • Attackers can exploit this mismatch
Evidence: The ASVspoof5 paper (§4.2) documents shortcut artifacts in prior databases. Müller et al. (2022) and ASVspoof5 results show significant performance degradation under codec mismatch — detectors overfit to codec-free artifacts and fail on compressed audio.

Why It Matters

  • Real-world audio is never pristine
  • Messaging apps, VoIP, and social media all apply compression
Goal: Build detectors robust to codec and channel shifts

ASVspoof 5: A Modern Benchmark

"We have been alerted to the presence of shortcut artefacts which plagues some earlier ASVspoof databases — namely artefacts which are semantically unrelated to the spoof/deepfake problem but which can nonetheless be utilised for detection."

— ASVspoof5 Paper

Split Samples Codec Diversity
Train ~180k Uncoded only
Dev ~140k Uncoded only
Eval ~700k Full codec diversity (C01–C11)

Train/Dev have no codec shift — the evaluation set is the true generalization test. This forces models to generalize to unseen transmission conditions.

Synthetic Codec Augmentation

Train/Dev have no codec diversity — we create it synthetically via FFmpeg:

Synthetic ASVspoof 5 Codecs Status
NONE "−" (original) Simulated
MP3, AAC C05, C06 Simulated
OPUS, SPEEX, AMR C01, C02, C03, C08, C09, C10 Simulated
— C04 (Encodec), C07 (cascade), C11 (device) Not simulated
Important — Transfer framing: Training uses synthetic codec labels (MP3/AAC). Evaluation uses real ASVspoof5 codec IDs (C01–C11). This is transfer of learned invariance to unseen real-world codecs — not direct supervised domain adaptation.

Augmentations are pre-computed offline for ~5× training speedup vs on-the-fly FFmpeg.

Key Observations

  • The gap reduction confirms DANN mitigates codec shortcuts
H1 supported: DANN reduces domain leakage and improves out-of-domain performance.

RQ3: Representation Analysis (CKA)

Layer-wise CKA Similarity: ERM vs DANN

  • Compare layer representations between ERM and DANN using Centered Kernel Alignment
  • High CKA = similar representations; Low CKA = DANN has diverged
  • Identify where DANN has the greatest impact on learned features

Key Findings

  • Early layers (0–6): CKA ≈ 1.0 (nearly identical)
  • Layers 7–10: gradual divergence
  • Layer 11: sharp CKA drop — maximum divergence
Interpretation: DANN's adversarial gradient primarily reshapes the final mixing layer, where domain information is most concentrated.
Note: CKA computed on WavLM representations. Low similarity at layer 11 reflects divergent learned mixing weights — backbone layers themselves are frozen and identical between ERM/DANN.

Domain Probe Findings

Linear probe accuracy predicting codec from frozen WavLM features:

Layer Accuracy Signal
0–2 85–87% Strong
3–4 72–80% Moderate
5–6 63–67% Weakening
7–11 ~60% Near-chance

Chance = 50% (binary codec classification)

Key Insights

  • Codec signal IS present — augmentation works
  • WavLM progressively removes it (87% → 60%)
  • ~10pp above chance remains at L11
  • DANN should remove this residual signal
Implication: SSL backbones achieve partial domain invariance through pretraining. DANN completes what the backbone started.

Future Work

Partial Unfreezing

  • Currently: SSL backbone is fully frozen
  • Proposed: Unfreeze top 2–3 transformer layers
  • Allow DANN's adversarial gradient to reshape backbone representations directly
Unfreezing the top 2–3 transformer layers would allow DANN's adversarial gradient to reshape backbone representations directly, potentially removing codec information at deeper levels. Given that CKA analysis shows layer 11 already diverges significantly in mixing weights alone, even partial unfreezing could yield substantial further invariance gains.

Other Directions

  • Activation patching (RQ4): Swap DANN activations into ERM at identified layers
  • More codecs: Add neural codecs (Encodec, SoundStream) to synthetic augmentation
  • Multi-seed evaluation: Confirm results across multiple training seeds
  • t-DCF evaluation: Measure real-world security impact in tandem with ASV

Summary

Key Findings

  • SSL backbones are critical — WavLM/W2V2 dramatically outperform classical baselines
  • DANN works — consistent improvement over ERM across both backbones
  • WavLM DANN achieves 7.04% EER — 14% relative improvement over ERM
  • Augmentation alone is insufficient — adversarial training is necessary
  • CKA confirms mechanism — DANN diverges most at final mixing layer

Contributions

  • First application of DANN to ASVspoof5 codec domain shift
  • Multi-head discriminator for joint CODEC + CODEC_Q invariance
  • Synthetic-to-real transfer of learned invariance
  • Comprehensive interpretability analysis (probes, CKA, t-SNE)
Answer to main RQ: Yes, detectors use codec shortcuts. DANN successfully mitigates this, improving OOD generalization.

All results from single training seeds. Bootstrap 95% CIs computed on eval set.

Bounding Confounds: Shortcut Classifier

Are shortcuts (duration, energy, silence) explaining the OOD gap?

Method: Train logistic regression using only 5 shortcut features (no audio content). If it works on train but fails on dev/eval → shortcuts are train-specific artifacts.

Split Accuracy AUC EER
Train 82.6% 0.903 17.7%
Dev 24.5% 0.451 53.6%
Eval 20.4% 0.483 52.7%

5 features from ASVspoof5 §4.2:
peak amplitude, energy, duration, leading/trailing silence

✓ Shortcuts are TRAIN-SPECIFIC
High train accuracy (82.6%) but random on dev/eval (~50% EER)
Conclusion: OOD gap is NOT primarily explained by these five shortcut artifacts (ASVspoof5 §4.2)

Novelty & Contributions

"Systematic evaluation of DANN for codec-robust deepfake detection, including transfer from synthetic to real codecs and representation-level analysis."
  • DANN applied specifically to codec mismatch problem
  • Frozen backbone + DANN = invariance in projection layer only
  • Probing + CKA analysis provides interpretability
  • Ablation confirms augmentation ≠ adversarial training
Practical takeaway: Domain invariance doesn't require fine-tuning billion-parameter models — a lightweight projection head is sufficient
